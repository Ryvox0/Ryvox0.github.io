Version Timeline

0.0.1

Initial Plugin Foundation

First public base for PulseEvents with the essential event flow and admin commands.

Added

  • Base plugin setup and startup loading
  • First event lifecycle manager
  • Core commands: start, stop, status
  • Simple announcements and chat feedback

Fixed

  • Startup registration issues for commands
  • Basic state reset after stopping an event
  • Safer checks for duplicate event starts
0.0.2

Countdown and BossBar Upgrade

Focused release that made events feel more alive before they started and more reliable while queued.

Added

  • Fake and in-game styled bossbar countdown concept
  • Queue handling for upcoming events
  • Improved actionbar and title messaging
  • More visible event status feedback for players

Fixed

  • Countdown desync after manual stop
  • Queue order issues when two events were triggered fast
  • Bossbar visibility not clearing correctly after finish
0.0.3

Fake Web Panel and Status Control

Version concept for showing where the plugin is heading: event list, statuses, and browser-side control ideas.

Added

  • Fake browser panel for changing plugin settings later
  • Interactive list of events with visible statuses
  • Start and stop controls in a panel layout
  • Landing page simulation with countdown and bossbar preview

Fixed

  • Stop-state messaging now resets cleanly in the demo
  • Selected event and panel state stay synchronized
  • Roadmap presentation now reflects version changes clearly
